## 一、项目起源

**作者身份**：双非大三学生，游戏策划方向。第一段实习在腾讯 WXG 做**平台产品策划（微信小游戏）**——非游戏策划。实习临近结束、未转正（主要是岗位错配，非能力问题），正在准备秋招。

**项目性质**：求职用的**作品集**Demo——重"设计深度 + 完整可玩的核心闭环"，不追求商业化。之前在 UE5 用 C++ 做过另一个作品《铸剑》（动作战斗原型）。

**灵感**："人们对锻刀大赛类节目很感兴趣，人类天生对冷兵器有需求感。" 同时受 **Noita** 影响很深——喜欢"提供一套规则系统，让玩家组合出无限可能"。

**初始方向**：微信小游戏平台，Cocos Creator 引擎。**后来经过讨论改为 PC 端为主（键鼠操控体验更好），但保留远期上微信小游戏的可能。**

## 二、设计演进时间线（重大转折点，按发生顺序）

### 第 1 阶段：原始构思
玩家扮演铁匠 → 锻造武器 → 武器带数值 → 分享给好友 → 好友拿着武器走一段"冒险故事" → 数值决定故事分支与结局。

**当时困惑**：锻造怎么让玩家"自己决定武器长啥样"？数值和故事怎么精密串联？用什么引擎？

### 第 2 阶段：锻造系统的第一性原理拆解
- **核心突破**：锻造不是"固定流水线步骤"，而是**"工作台状态机"**——台上始终有一块材料对象，带一组状态（温度/形态/层数/成分…），玩家自由排列操作，操作有合法门槛。大马士革钢 = 把"敲薄→锯→叠→熔"循环 N 次，同一套原子操作不同排列，不动地基。
- **操作拟真方向**：最初是触屏"点/拖/滑"三手势，后来作者提出要**更强的实机锻造感**——类似《锻刀大赛》的操作体验
- **平台演进**：最初定为微信小游戏（触屏），→ 后改为 **PC 键鼠为主**(Demo 验证最带感，键鼠精度更高)，核心逻辑与输入层解耦、保留手机/微信触屏适配的可能。

### 第 3 阶段：数值体系的深挖（多次推翻重来）
- **初版 6 维**：锋利/硬度/韧性/重量/外观/平衡。用一个 HTML 探路原型验证时，发现**致命漏洞**——"除了重量越低越好，其他全是越高越好"→**没有取舍就没有 Build，没有你要的'组合出无限可能'。**
- **修正为"三对物理取舍"**：源自真实冶金——硬度↔韧性（淬火越硬越脆）、锋利↔刃口耐用（越薄越易崩）、轻↔结实。**无全满神刀。**
- **新增"回火"步骤**（重大发现）：真实世界"硬↔韧"的取舍靠淬火后的回火温度来调（温度越高越韧越软）。游戏里做成"玩家拖动回火温度条"——把取舍变成**玩家的主动决策**，比淬火自动决定高级得多。
- **耐久内化**：由硬度+韧性派生，不单开第 7 维（保持简洁）。
- **6 维的真正定位**（重要澄清）：不是"最根基属性"，是"**4 个根基物理量（成分/微观结构/几何/内部完整性）的计算输出 + 面向故事的压缩接口**"。根基物理量靠 Powder Toy/Noita 式模拟——但这是**远期愿景，放 Demo 之后**，Demo 用简化取舍公式。
- **研磨 = 第二塑形阶段**（纠正）：真实中"刀能不能切靠磨出的刃面斜角"，所以研磨不只是开刃，它能改轮廓、减薄、定刃角，是"锋利↔耐用"取舍的操作点。

### 第 4 阶段：卖点二的定位演变（从"判定分支"到"轻跑团"）
- **初期**：数值→固定判定表→不同故事分支。用探路原型验证后，暴露"不够有趣"的问题。
- **作者提出要做成"跑团"**（选择+武器+世界状态影响结局）——**方向正确但极度危险。** 真跑团（剧情大分叉）是内容黑洞——每多一个选择就多一条独立剧情线，分支指数爆炸，一人必烂尾。
- **最终定为"轻跑团"**：用少量**世界状态变量**（名声/结仇/体力…）+ 条件文案。选择改变状态变量，后续节点读取"变量+武器数值"读出不同文案，**主干不真分叉**。用《这是我的战争》的方式达到"千人千面"体感。
- **无通用最优解**（关键设计）：不同冒险线**考验完全不同的数值组合**（潜行线爱轻+锋利、攻坚线爱重+硬度），"你想去哪冒险"决定"你该造什么武器"——保护探索欲。
- **关键时序决策**：卖点二的完整设计**推迟到锻造(S3)做完、有真实武器数值后再做**——因为脱离真武器设计跑团=空想（假数值必定被真实分布推翻）。

### 第 5 阶段：美术与系统的"割裂"→"缝合"（一次重大纠偏）
- **偏差**：早期"武器部件库"方案——25 个手枪械素材拼出 1400+ 武器——那是从"省美术工时"出发的方案。但它把玩法从**"锻造创造"降级成了"选零件拼装"**，与"玩家自己决定武器长啥样"冲突。且美术方案是独立于系统之外的，**没有长在系统上。**
- **纠正 = 物理外观两层模型**：
  - **第一层·系统实时生成**（形状/温度色/光泽）：由锻造物理状态直接驱动，引擎实时算。**几乎不需美术，保证"敲一下就变"。**
  - **第二层·工艺涌现分级贴图**（大马士革水纹/包钢分层线等）：只有当物理量达到某工艺阈值，**系统客观判定"某工艺达标"**，才叠加对应**分等级**的预制贴图。——这**不是玩家选的，是玩家锻出来的。**
  - **美术清单 = 系统工艺种类 × 等级数**——需求从系统长出来，不再割裂。
  - **基础形制起手 + 锻造自由塑形**：玩家选刀/剑/斧等基础形制起手（这是第二层贴图对齐的骨架锚点，贴图贴在鬼形状上会错乱），锻造自由决定具体形状/厚薄/材质/数值。
- **美术基调**：小丑牌(Balatro)风格（精致像素+厚描边+鲜明暖色+讲究光影）。AI 做此精致度有难度，**先做试做样例验证**，达不到则退回"扁平矢量+厚描边+暖色光影"。

### 第 6 阶段：技术壁垒诚实评估
- **唯一真技术壁垒**：锻打/研磨的实时形变可视化。其余（回火/淬火/正火/翻车判定）全是纯逻辑、AI 最擅长、零壁垒。
- **跨壁垒方式**：用"分段缩放"（刀坯切 8-12 段，敲哪段哪段变粗细/长短）——效果足够，Cocos 基础能力。**不做真实物理仿真（C 方案，3A 级别，永不做）。**
- **关键认知**：手感 70% 来自反馈（火花/震动/音效/数字跳动），不来自形变精度。S5 重点是调反馈爽度。
- **"拟真↔游戏"是可滑动旋钮、非二选一**——也是技术保险：任何一步做不动，就往游戏端（离散化、简化）滑一格降低技术难度，不丢核心乐趣。
- **"能离散就不连续"**：旋转固定 90°（4 个面），把操控地狱降成可控组合，几乎不损失深度。

### 第 7 阶段：开发方式确定
- 作者**不碰代码**，用 Prompt 工程指挥 AI，做 "AI 架构师 + 质检员"。
- **不用多 Agent 团队**（现阶段单人单线、成本 > 收益，触发时机 = 内容爆发期）。
- 规范流程：**轻量分支流**（main 永远可跑，每模块 `feat/` 分支，验收通过才 --no-ff 合并）+ **每步验收=一次 commit** + 修改前先说计划等点头。
- 跨设备接力：一切靠 git 仓库（GitHub: meteoryzx/Weapon-Forger），不靠对话记录。换设备=clone + 读 START_HERE。

## 三、作者的设计哲学（理解这一条，就理解了所有决定）

> **受 Noita 影响："提供一套规则系统，让玩家组合出无限可能。"**

这体现在：
1. **锻造** = 操作为地基、工艺为配方——不写"包钢=+5"，让优劣从规律中涌现
2. **故事** = 6 维为接口、内容为配方——加 DLC 只塞情境/碎片，系统不动
3. **无最优解**——不同冒险线考验不同数值组合，不同武器各有战场
4. **深度来自组合、不来自单步难度**——易学难精

---

## 四、几个容易踩的坑（作者反复强调的）

1. **别在纸上拍死"具体数值/故事分支"**——必被原型推翻，去原型里看着反馈定
2. **别把"做不完的完美方案"当成目标**——这是作品集，宁可小而完整、绝不大而烂尾
3. **卖点二的叙事设计别脱离真实武器**——必须等锻造做完、有了真实数值分布，才设计具体的冒险线和分支
4. **美术方案要长在系统上**——不要搞"和系统独立的素材库"
5. **变量/数字/文案全部进 data 表**——作者要能自己改，不求 AI
