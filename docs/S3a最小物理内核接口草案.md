# S3a 最小物理内核接口草案

> 状态：只定义下一切片的边界与验收，尚未实现。实际类型、参数和测试只能在 S0 Cocos/Node 工程验收后进入 `assets/scripts/forge/`。

## 这一步要证明什么

同一根连续坯料在不同温度、受击方向、受击位置和锤击能量下，产生稳定、可解释且可回放的不同状态。它不是软体物理，也不是“点击后直接给六维加分”。

S3a 不做 Cocos 输入、网格、粒子、音效、淬火 UI 或最终六维。它只生产能被之后的渲染、评估和故事读取的施工状态。

## 数据边界

```text
ForgeOperation[] + ForgeParameters + 初始材料
                         |
                         v
                    ForgeState
                    /          \
        ForgeSnapshot            S3c/M3 evaluation
        （给渲染）                 （最终 WeaponData）
```

### ForgeState

完整施工真相，必须可序列化、可回放。至少包含：

- `version`：状态与参数版本，防止旧存档被悄悄按新规则解释。
- `seed`：仅给明确需要随机的扩展内容；S3a 锻造结果本身不得依赖随机。
- `workpieces`：当前材料块数组；首版一块，接口允许未来多材料。
- `operations`：已接受的不可变操作记录。
- `phase`：当前工序阶段，用来拒绝不合法操作但不替玩家决定目标形状。

### WorkpieceState

每块材料的材料学与几何状态：

- `materialId`、`mass`、`temperature`、`microstructure`、`integrity`。
- `sections`：8-10 个 `BladeSection`，按从柄端到尖端的固定顺序保存。
- `joints`：当前为空，预留给 S3d 装配和未来焊接。

### BladeSection

局部截面是首版连续坯料的最小单位。至少保存：

- `position`、`length`：沿刀坯方向的位置与段长。
- `width`、`thickness`：当前截面的两条正交尺寸。
- `temperature`、`plasticity`、`stress`、`integrity`：局部热与损伤状态。
- `lateralOffset`：单侧锻打引起的局部偏移，最后可推导弯曲。

所有截面共同承担近似体积守恒：锤击使局部厚度/宽度减少时，长度或邻接段的尺寸必须出现对应变化。首版只追求可解释趋势，不追求工业级精度。

### JointState

为受控装配和未来焊接留出结构，不开放任意焊接 UI：

- `id`、`kind`、`workpieceIds`、`location`、`contactArea`、`quality`、`integrity`。

### ForgeOperation

每次玩家意图必须变成可回放操作，不直接修改评估数值：

- `heat`：目标温度或热源强度与持续时间。
- `rotate`：`quarterTurns`，只允许 90 度离散旋转。
- `hammer`：目标截面、当前面、归一化能量、落点偏移。
- `quench`：介质与浸入时长。
- `grind`：目标截面、侧面、移除量与刃角意图。

S3a 实现 `heat`、`rotate`、`hammer` 的纯逻辑；`quench` 和 `grind` 只预留联合类型，不写规则。

### ForgeSnapshot

渲染用的只读投影，不保存为施工真相：

- 每段的中心线、宽度、厚度、温度色、完整性和裂纹标记。
- 当前操作位置与受击面。
- 全局形状摘要，例如总长度、体积误差、弯曲量和重心。

渲染层只能读它。Cocos 节点变换、粒子和声音绝不能回写 `ForgeState`。

## S3a 首版规则与测试样本

| 规则 | 玩家可感结果 | 自动样本 |
|---|---|---|
| 温度 -> 塑性 | 热坯比冷坯更容易有效形变 | 同能量锤击：高温厚度变化更大 |
| 低温锤击 -> 应力 | 冷敲更危险 | 同能量锤击：低温应力/损伤更高 |
| 四面受击 | 转面后打出的宽厚差异正确 | 90 度旋转后宽度与厚度响应交换 |
| 邻域影响 | 不出现孤立断层 | 当前段最大变化，邻段平滑衰减 |
| 近似体积守恒 | 打薄不会凭空消失材料 | 标准序列的体积误差保持阈值内 |
| 单侧受击 -> 弯曲 | 偏打一边会歪 | 相反两侧抵消，单侧累计偏移 |
| 确定性裂纹 | 风险可理解、可重现 | 同状态同操作永远在同段产生同裂纹 |
| 回放 | bug 可以复现 | 相同初态与操作 JSON 得到深度相等结果 |

## 明确不做

- 不做软体、有限元、粒子材料或真实冶金求解。
- 不直接输出六维、总评分或“成功/失败”。
- 不做任意焊接、折叠钢、大马士革、正火、回火、完整淬火和研磨规则。
- 不做 Cocos 节点、网格、输入或 UI；这些属于 S3b/S3c。

## S3a 完成定义

1. 纯 TypeScript 测试可在没有 Cocos 编辑器的环境运行。
2. 四种标准样本和一次 JSON 回放均可重复。
3. 状态接口不 import Cocos，快照不允许反向写模拟。
4. PR 给出一份可读的样本对比输出；作者能看懂“为什么两次锻造不同”。
5. Codex review 后，作者确认因果趋势可信，再进入 S3b。
