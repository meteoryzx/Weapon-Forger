# 第一部分 · 项目定位

## 一句话定位
> **你是一名铁匠，用可自由组合的锻造工艺打造武器；每把武器带着它的 6 维数值，替你或好友走进一段"由数值决定分支与结局"的冒险故事。**

**两大卖点**：① 自由、拟真、可重复的锻造过程；② 数值驱动的有趣冒险故事（可分享给微信好友）。

**设计哲学**（贯穿全局，受 Noita 影响）：**"提供一套规则系统，让玩家组合出无限可能"**。锻造是"操作为地基、工艺为配方"；故事是"6 维为接口、内容为配方"——两者同构，全局自洽。

**项目性质**：作者求职**作品集**（目标：游戏策划岗）。重"设计深度 + 完整可玩的核心闭环"，不追求商业化完整度。引擎 Cocos Creator；**Demo 阶段导出 H5 网页版验证（浏览器直接玩、发链接演示），暂不碰微信开发者工具**；远期目标平台微信小游戏（上线时再用 Cocos 导小游戏版）。

**Demo 范围（务必控制，防膨胀）**：教学(打第一把普通剑) → 放养自由锻造 → 出 6 维武器 → 两个冒险出口〔雇 NPC 自己看 / 二维码分享好友看〕→ 结局。
**暂不做**：沙盒装饰购买、多 NPC 长线、大马士革全工艺、分级美术全库、真实服务端。

# 第二部分 · 玩法与系统设计

> 标记：`✅ 已定` / `🔶 挂起（留后续）` / `🔴 待验证`。

## 1. 玩家体验循环
```
① 锻造：自由排列操作 → 产出一把带 6 维数值 + 外形的武器
② 交付（两个出口）：
   ├─ 雇 NPC 带去冒险 → 玩家自己看故事（即时反馈，解决"铁匠看不到高潮"）
   └─ 生成二维码分享好友 → 好友看故事（社交传播）
③ 成长：冒险回流资源 → 升级材料/工具、解锁图纸 → 造更强/更炫的武器
```
> **关键结构**：双出口保证铁匠玩家有即时爽点，不把留存全押在好友反馈上。

## 2. 锻造系统（卖点一）

### 2.1 系统本质：工作台状态机（非固定流水线）✅
台上永远有一块「材料对象」，带一组状态（形态/层数/成分/温度…）。玩家从操作面板选一个**当前合法**的操作施加，材料状态随之改变。**没有"第几步"，只有"当前是什么状态、允许做什么"。**
- 常规剑：`选材 → 熔炼 → 敲打 → 淬火 → 打磨 → 装配`
- 大马士革：`… → 敲打成薄片 → 锯开 → 堆叠 → 再熔炼 →（循环N次）→ 敲打 → 淬火 → …`
- **同一套原子操作、不同排列**。未来任何新工艺 = 新操作 + 新门槛，不动地基。

**操作合法性门槛（示例）**：
| 操作 | 合法前提 | 做完后 |
|---|---|---|
| 熔炼成锭 | 碎料/薄片堆 | 热锭（形态重置，层数累加保留） |
| 敲打成薄片 | 锭且够热 | 薄片（可被锯） |
| 锯开 | 薄片 | 2+ 块薄片 |
| 堆叠 | 有 2+ 块薄片 | 一叠（层数=各层之和） |
| 淬火 | 已成刀形且够热 | 定型，硬度锁定 |

### 2.2 五原子操作 × 三手势 ✅
核心交互只三种手势：**点（敲）/ 拖（淬火·取料）/ 滑（打磨·折叠）**。
| 原子操作 | 手势 | 改变的底层状态 |
|---|---|---|
| 加热 | 按住/拖控温条停在锻造窗口 | 可塑性（临时） |
| 施力·敲打 | 点击落锤位置 | 形态（局部厚度/长度） |
| 施力·折叠 | 长按+对折 | 微观结构（层数↑） |
| 冷却·淬火 | 正确时机拖入水/油 | 微观结构（硬度↑，过头脆） |
| 增减材料 | 拖拽矿料/覆土/合金 | 成分 |
| 研磨 | 沿刃口来回滑动 | 表面（锋利/外观） |

底层状态四类：**① 形态 ② 微观结构 ③ 成分 ④ 表面**。任何工艺都是对这四类的操作组合。

### 2.3 游戏结构：2.5D 场景中枢 + 工序小游戏 ✅
可沙盒摆放的 2.5D 铁匠铺为**导航中枢**，每件工具 = 一个操作入口：
```
2.5D 铁匠铺（斜俯视）
  材料架→选材 / 熔炉→加热 / 铁砧→锻打 / 淬火槽→淬火 / 锯台→锯折 / 磨石→打磨 / 装配台→装配
  点击入口 → 进工序小游戏 → 完成返回 → 点下一个
```
- 场景里"可点亮"的工具 = 当前材料合法的操作（与状态机一致）。
- 🔴 **双视角注意**：场景斜俯视，工序（尤其锻打/打磨）需正侧视看形变。点入口做"推近/切镜"过渡；**两套镜头美术要分开规划**。

### 2.4 "透明的自由"准则 ✅
自由 ≠ 黑箱瞎试。必须：实时状态可视化 + 操作即时反馈 + 面向武器的约束（只保留对武器有意义的操作）+ 失败可读（裂纹时提示"层数太高/材料太差"）。
**核心平衡：深度来自"操作序列的组合"，不来自"单步难度"（易学难精）。**

### 2.5 锻打/打磨技术方案（唯一真难点，已论证可行）✅
**原则：参数化伪形变，绝不做真实物理仿真。** 玩家要的是"我砸这里→这里真变了"的因果感。
- **锻打**：刀坯横切 8–12 段，每段带「厚度+长度偏移」；敲某段→变薄向两侧延展→按参数实时重绘轮廓。实现：分段网格变形（推荐）或分段贴图缩放。这组参数**就是 §3.4 的 WeaponShape**（锻打=外形=数值来源，三位一体）。
- **打磨**：刃口盖灰暗遮罩，滑动擦除露出锋利反光；覆盖率=锋利、均匀度=外观、漏磨=暗斑。技术=遮罩擦除（同刮刮卡）。
- **结论**：其余工序都是"改参数换画面"零难度；唯一需调校的是**锻打网格变形的手感**——成熟方案，属施工验证项，非漏洞。

### 2.6 如何评判玩家操作"好不好"（🔴 重要待决，原型中定）
> 这是锻造系统的隐藏地基。若判错，会毁掉"自由 + 滑稽翻车"的核心乐趣。**具体规则留到 S2/S3/S5 原型里定，但方向如下（建议，可改）**：

**拆成两个独立的轴，不能混为一谈**：
| 轴 | 判什么 | 游戏态度 |
|---|---|---|
| ① **操作精度**（执行得好不好） | 淬火时机是否卡窗口、敲击是否落有效区、打磨是否均匀 | **有客观对错** → 做得好=数值上限高、成功率高（手感与成就感来源） |
| ② **意图达成**（做的是不是你想要的） | 成品符不符合玩家自己的目标 | **无价值判断** → 想做锋利匕首就看锋利，想做搞怪的就无所谓（自由与滑稽感来源） |

**关键**：游戏只对 ①精度 给"好/坏"反馈，对 ②意图 不做评判。三层评判信号：
- **过程即时反馈**：每次操作当下就给"漂亮/一般/搞砸"（火花大小、音效、"完美！"字样）；
- **成品数值**：6 维等级 = 客观成绩单；
- **最终交给冒险验货**：一把精度差的剑，冒险故事里就真的会崩会断——**把"好不好"的最终裁决权交给玩家最在乎的环节**。

> 一句话：**精度可评判，意图不评判，最终由冒险故事"用后果说话"。**
> 🔴 待原型确定：每个操作的精度判定窗口大小、精度→数值的映射曲线、即时反馈的强度。

## 3. 武器评估体系（三层结构：物理模拟 → 6维评估 → 故事）

### 3.0 整体结构（本项目数值哲学，先读）✅
```
【物理模拟层】4 个根基物理量  → 决定一切的客观底层
  成分(含碳/合金) · 微观结构(晶粒/相/层数) · 几何形态(刃角/厚度/配重) · 内部完整性(内应力/裂纹/夹杂)
        ↓ 由公式计算（客观、涌现，不人为设定"某工艺=+X分"）
【评估层】6 维数值(0–10)      → 面向玩家和故事的"压缩翻译层"
        ↓ 分段触发
【故事层】数值 → 分支 → 结局   → 故事"生长"出来
```
- **核心理念（受 Powder Toy/Noita 启发）**：不写"包钢=+5"这种人为规则，而是**模拟根基物理量**，让"包钢更好/大马士革更韧"这些结论**从模拟中自动涌现**——规则客观、工艺真实。
- **6 维的准确定位**：不是"最根基属性"，而是**根基 4 量的计算输出 + 面向故事的压缩接口**。它的使命不是"完整描述一把刀"，而是"用最少维度抓住能生长出故事的特征"。
- **落地节奏（重要）**：物理模拟是高投入项，**放 Demo 之后**。Demo 阶段 M3 数值模块先用"简化取舍公式"（§3.2），验证好玩后再把 M3 内部升级为物理模拟——**WeaponData 接口不变，锻造/故事两边不动**（架构解耦为此留了口子）。

### 3.1 6 维数值（等级制 0–10，评估层）✅
| 维度 | 含义 | 由哪些根基量决定 | 冒险中被判定的场景 |
|---|---|---|---|
| 锋利 | 切割/伤害 | 几何(刃角) + 硬度 | 战斗、砍断障碍 |
| 硬度 | 不易变形 | 成分(碳) + 微观结构(淬火) | 硬碰硬、破甲 |
| 韧性 | 不易折断 | 微观结构(晶粒/层数) + 内部完整性 | 格挡、久战、恶劣环境 |
| 重量 | 轻重手感 | 几何 + 成分(密度) | 敏捷、跋涉、偷袭 |
| 外观 | 观感/威慑 | 微观结构(纹理) + 几何 + 表面 | 社交、威慑、身份误判 |
| 平衡 | 综合协调 | 几何(配重) | 高难连续判定、特殊结局门槛 |

- **完备性（严格结论）**：作为"物理性能"评估层**基本完备**——一把刀的物理性评价几乎都能由 6 维派生。
- **显示 = 等级制**：玩家看 0–10 等级，不看裸数字。

### 3.1b 三对物理取舍（源自真实冶金，破解"越高越好"）✅
6 维不是 6 条独立能力值，而是**几组互相拉扯的取舍轴**——物理上无法全满：
| 对立 | 原理 | 游戏内简化 |
|---|---|---|
| 硬度 ↔ 韧性 | 淬火越狠越硬但越脆 | 狠淬火 = 硬度+ / 韧性− |
| 锋利 ↔ 刃口耐用 | 刃越薄越利但越易崩卷 | 精磨 = 锋利+ / 耐用− |
| 轻 ↔ 结实攻坚 | 越轻越灵活但越不抗造 | 减料 = 轻+ / 结实− |
- **平衡/外观**不参与对立，是"综合工艺水平的奖励"，手艺越好整体越高。
- **手艺的意义**：不是"无脑全满"（物理不允许），而是"**更精准地把资源分配到你想要的那一端 + 减少损耗**"。
- **特殊工艺（图纸）= 缓解对立**：如大马士革让"硬度+3 时韧性只−0.5"——花手艺/材料换来"两头都保住一点"，这就是高级工艺的价值与追求动力。

### 3.2 派生值与"耐久"内化 ✅
- 暴击/破甲/挥速/威慑/价值等 = 由 6 维派生，不单列。
- **耐久（持久度）= 由 硬度+韧性 派生**（脆的刀耐久低），**内化不单开第 7 维**，保持简洁。🔶 若后续发现耐久特别能出故事，再评估提为正式维度。
- Demo 阶段 M3 用"简化取舍公式"实现上表关系（§3.1b），未来升级为物理模拟。

### 3.3 特殊功能属另一支，不塞进 6 维 ✅
"淬毒 / 引火 / 认主"等**非物理的特殊/魔法功能**，无法也不该从物理 6 维派生 → 由 **工艺标签 / 图纸 / 故事层** 承载。
> 一把刀的完整评价 = **物理性能支（6 维）+ 特殊功能支（工艺标签）**，两支合起来才完整，架构上已分开。

### 3.4 总评分 ✅
6 维加权+工艺完成度，**仅用于**：① 决定套哪档品质贴图；② 社交门面数字。**不参与冒险判定**（判定仍逐维判，保偏科翻车戏剧性）。🔶 加权公式待数值阶段定。

### 3.5 外形数据 WeaponShape ✅
外形**不存预制图，存锻造产生的参数**：`{基础形制ID, 刀身控制点[], 各段厚度数组[], 材质ID, 工艺标签[], 表面状态}`。这组参数由玩家锻造实时写入，同时喂渲染层、数值层、判定层。
> **关键澄清（纠正过一次偏差）**：武器外观**不是"从预制部件库里选零件拼装"，而是玩家用锻造系统"打"出来的**——敲打改形状、淬火改颜色、折叠改纹理。外观是**锻造的产物**，不是**选择的素材**。这如何在美术上落地，见 §3.6「物理外观两层模型」。

### 3.6 物理外观两层模型（美术与系统的缝合点，⭐核心）✅
> 用第一性原理拆：真实世界一把刀的外观只有两个来源——**连续变化的物理量** + **达标才涌现的工艺特征**。据此外观分两层叠加，**让美术"长在系统上"，而非独立供货**：

**第一层 · 系统实时生成（连续层，无需美术预制）**
由锻造物理状态直接驱动，引擎实时算：
| 表现 | 由什么驱动 | 实现 |
|---|---|---|
| 形状（弯直厚薄） | WeaponShape 控制点/厚度数组 | 网格变形实时重绘 |
| 基础色 | 温度/含碳量/氧化程度 | 颜色映射实时取色 |
| 表面光泽 | 打磨覆盖率 | 遮罩 + 高光强度 |
→ 保证"**你敲的每一下都立刻反映在画面上**"（创造感/留痕感）。

**第二层 · 工艺判定触发分级贴图（涌现层，用预制美术）**
当底层物理量达到某工艺阈值，系统**客观判定"某工艺达标"**，叠加对应的、**分等级**的预制贴图：
| 触发条件（物理量达标） | 涌现的外观 | 贴图分级 |
|---|---|---|
| 层数≥阈值 且 均匀度高 | 大马士革水纹 | 工艺越精湛套越高档纹理 |
| 分层结构(硬背韧刃) | 包钢分层线 | 同上 |
| 抛光度≥阈值 | 镜面光泽 | 同上 |
→ 这是"**美术精致度**"（小丑牌质感）的来源，但**不是玩家选的，是玩家锻出来的**（系统判定触发）。

**为什么这样就"全然一体"了**：
- 美术资产清单 **= 系统里可涌现的工艺种类 × 每种等级数** —— 美术需求从系统长出来，不再拍脑袋、不再割裂；
- 玩家锻造行为**直接决定看到哪张贴图**，美术是"锻造的结果展示"而非"选装组件"；
- 精致部分（工艺贴图）预制手绘=可以很美；实时变化部分（形状/温度色）shader 生成=够用即可——**两者各司其职，不再打架**。
- **基础形制起手 + 锻造自由塑形**：玩家选一个基础形制（刀/剑/斧）起手，锻造决定其具体形状/厚薄/材质/数值。基础形制是"第二层贴图能对齐"的技术锚点（水纹要贴得上，需一个大致骨架），**这是技术必需，非妥协**。（技术接口见 `AI工作规范.md` §3、M6）

## 4. 数值 → 故事生成体系（卖点二核心引擎）

### 4.1 故事"好看"的四源泉
| 源泉 | 本质 | 如何驱动 |
|---|---|---|
| ① 反差/意外 | 结果超预期 | "外观9锋利1"唬住强盗却砍不动柴 |
| ② 因果/掌控 | "我造成的" | 自己淬火手抖致剑脆，冒险中真断了 |
| ③ 代入/情感 | 在乎主角 | 真人好友带着这把剑，玩家在乎害不害 TA |
| ④ 变数/稀有 | 罕见惊喜 | 极端数值触发隐藏结局，炫耀传播 |
①②④ 数值驱动，③ 社交驱动。

### 4.1b 探路原型结论（2026-07-15，HTML 验证）
做了"数值→故事"HTML 探路原型（`探路原型/数值到故事_原型.html`，验证即弃、不进正式代码）。**收获的结论**：暴露出旧 6 维"除重量外越高越好、无取舍"的骨架问题 → 促成 §3.1b 三对取舍修正。验证了"数值驱动故事分支"这条链在交互上成立，为 S2 正式开发提供参照。

### 4.2 三层生成结构（可无限扩展、可加 DLC）✅
```
情境库(可无限加) × 6维分段(固定接口) → 叙事碎片库(可无限加)
"遇到强盗"        外观[0-2/3-5/6-8/9-10]  每个交叉格=一段文字
"过腐烂的桥"      重量[分档]              组合生成故事
"黑暗森林"(未来)   韧性[分档]
```
- 一个节点故事 = 情境 × 被考验维度 × 落在哪段 → 对应叙事碎片。
- 加 DLC = 塞情境/碎片，6 维系统一行不改。

### 4.3 判定节点示例（数据驱动配置表）
| 节点 | 判定维度 | 分段 → 分支 |
|---|---|---|
| 遇强盗 | 外观 | 0–3 放行 / 4–8 战斗 / 9–10 敬畏放行 |
| 过朽桥 | 重量 | 高→桥断落水 / 低→安全 |
| 恶战 | 锋利+硬度+韧性 | 综合达标→胜 / 否→伤退 |
| 神庙门(隐藏结局) | 外观+平衡 门槛 | 全达标→隐藏结局 / 否→普通结局 |

### 4.4 冒险难度设计 ✅
**"难" = 结局稀有度，非通关难度**：大部分冒险正常武器能走完；稀有/隐藏结局才设硬门槛。玩家追求"解锁更炫结局"。单点极端会翻车 → 戏剧性。

## 5. 成长体系：材料 × 工具 + NPC 教学

### 5.1 操作自由，"做多好"受材料/工具限制 ✅
- 任何操作都能做（自由度拉满）；高级工艺**能否成功**取决于**材料等级 + 工具等级**。
- **步骤外化为工具**：拥有工具 = 能把某操作做更好 → 天然金币出口 + 可视化成长。
- **工具 = 精进式**（提升质量上限，不锁操作）：徒手能做，工具越好该工序上限越高。

### 5.2 失败即内容（裂纹判定）✅
```
每次堆叠/折叠：成功率 = f(材料等级, 工具等级, 当前层数)
层数越高/材料越差/工具越烂 → 越易失败 → 材料打"裂纹" → 硬度/韧性↓ + 视觉裂痕
```
不禁止尝试，材料不够就翻车出裂纹 → 失败变笑料与故事素材。想做完美 → 升级动力。

### 5.3 NPC 任务 = 教学 + 图纸 + 解锁 + 隐藏钩子（四合一）✅
- **逐步教学**：前几个 NPC = 伪装成剧情的新手引导。
- **图纸 = 可无视的"新手拐杖"，非工艺锁**：图纸只是简化工艺的提示卡；懂行者可无视自由发挥。高级任务给图纸（大马士革/百炼钢/灌钢/包钢等）。
- **未记载工艺反馈 = 方案 C**：自由发挥效果**如实反映在数值/外观**（即时回报）；但"命名/图鉴"只给图纸记载的（收藏党）。理由：小游戏非硬核拟真，免做工艺自动识别库。
- **解锁循环**：NPC 奖励+金币 → 解锁新材料/工具 → 做更高级工艺 → 探索欲驱动。

## 6. 社交系统（卖点二传播闭环）

### 6.1 双出口 ✅
- **雇 NPC 冒险自己看**：即时反馈，铁匠核心爽点。
- **分享好友看**：武器做完 → 6 维评级 → 结果此刻已定 → 冒险故事做成 PPT 式分段 → 生成二维码。好友扫码看一段已定但对 TA 未知的冒险（保悬念）。

### 6.2 分享技术思路 ✅
二维码指向轻量 H5，URL 带 **6 维参数**，页面用同一套判定表**现场跑一遍**生成故事——不存整段故事，只传 6 个数字。demo 阶段可先"本地看故事"，分享最后接。

### 6.3 社交双引擎 ✅
- **搞怪向**（即时传播）：搞怪武器触发滑稽故事（对应"反差"）。
- **收集向**（好奇/留存）：精心打造看能否走到结局；**冒险途中须"暗示"隐藏结局存在**（如神庙旁白），撺掇好友催铁匠再打 → 回流。此暗示须写进故事生成规则。

🔴 待验证（完整版）：好友无微信降级体验、故事可否再分享、服务端防篡改。**Demo 用本地模拟规避。**

# 第三部分 · 美术方案

## 总基调（所有图都遵守）✅ 已定档
> **小丑牌（Balatro）式风格：略带像素颗粒 + 相对精致 + 厚描边 + 鲜明暖色 + 讲究的光影。**
> 追求"看似像素、实则精致"的手工质感（像 Balatro 的卡牌）；不走纯低保真像素（太糙），也不走纯扁平矢量（太轻）。
> 精致感来源 = **像素/描边的手作质感 + 讲究的光照高光 + 统一后处理**，不是靠逐帧堆量。
> AI 关键词：`Balatro style, refined pixel art with clean outlines, painterly pixel, warm vivid palette, crisp lighting and highlights, tarot-card craftsmanship, cozy fantasy, high quality game art`

> ⚠️ **风险提示（务必记住）**：小丑牌的精致是专业美术逐张打磨的，AI 直接产出很难一次到位。**先做试做样例验证 AI 能不能稳定产出这种质感**（见下）；若达不到，退而求其次用"扁平矢量 + 厚描边 + 暖色光影"（成本更可控）。**风格统一 > 单张精美**——这条比像不像小丑牌更重要。

**统一风格锚点流程（重要）**：先只生成 **1 张最满意的"样张"**（如一把剑），当作**风格参照图**，之后所有资产都带它做 style reference。
> 说明：作者会尝试多种美术 agent，本方案以"规格与验收标准"为准，不绑定具体工具。

## 资产五大类
> **核心前提**：武器外观走 §3.6「物理外观两层模型」——第一层（形状/温度色/光泽）由**系统实时生成、几乎不需美术**；美术真正要产的是**第二层"工艺涌现贴图" + 基础形制底图**。所以下面 A 类不是"部件拼装素材库"，而是"**系统渲染要调用的分级贴图资产**"。
| 类 | 资产 | 说明 | 难度 |
|---|---|---|---|
| A | 武器渲染资产（基础形制底图 + 工艺分级贴图） | 供系统两层模型调用，**美术清单由系统工艺种类决定** | ⭐⭐ |
| B | 锻造场景+工具 | 卖点一舞台 | ⭐⭐ |
| C | 冒险场景（剪影分层） | 卖点二舞台，量大 | ⭐ |
| D | 角色/NPC/怪物（剪影） | 冒险对象 | ⭐ |
| E | UI/图标/特效 | 界面反馈 | ⭐⭐ |

### A · 武器渲染资产（配合两层模型，非拼装库）
武器外观 = 系统实时层 + 工艺贴图层（见 §3.6）。美术只需产**两类东西**：

**A1 · 基础形制底图（第二层的对齐锚点）**
| 形制 | 首发数 | 说明 |
|---|---|---|
| 基础形制 | 4–6 | 刀 / 剑 / 匕首 / 斧 / 巨剑…玩家起手选，锻造在其上塑形 |
- 作用：给"实时形变"和"工艺贴图"一个大致骨架，别让锻造出的形状太天马行空导致贴图错乱。
- 规格：可被网格变形拉伸的干净底图（分段友好）、透明背景、统一视角与描边。

**A2 · 工艺涌现分级贴图（第二层，美术精致度所在）**
| 工艺贴图 | 分级 | 触发条件（系统判定） |
|---|---|---|
| 大马士革水纹 | 3–4 档（越精湛越华丽） | 层数高 + 均匀度高 |
| 包钢分层线 | 2–3 档 | 硬背韧刃分层结构 |
| 镜面/抛光光泽 | 2–3 档 | 抛光度达标 |
| 材质基底(粗铁/精钢/玉钢…) | 每种 1 张 | 材料 + 工艺水平 |
> **美术数量 = 工艺种类 × 等级档数**（从系统里长出来，不是拍脑袋）。首发工艺少、每种几档即可。
- **用法**：`WeaponData` 的 `process` 工艺标签 + 物理量 → 系统判定该套哪档贴图 → 叠加到实时生成的刀身上。玩家锻得越好 → 触发越高档贴图。

> ❌ **已废弃的旧方案**：早期"刀身/护手/柄拆图层槽、25个素材拼1440把"的**部件拼装库**——它把玩法从"锻造创造"降级成了"选零件"，与核心诉求冲突，已被两层模型取代（见 DECISION_LOG G3）。

### B · 锻造场景+工具
铁匠铺背景(暗底,炉火作主光源)、熔炉(发光核心/体积光源)、铁砧、淬火桶、磨石、工具(锤/钳/锯，每种 3–4 级越高越精致)。规格：暗底让暖光成焦点、各物件单独出图、透明背景。

### C · 冒险场景（剪影三层景深）
每个场景 = **前景纯黑剪影 + 中景主体 + 远景(雾化/渐变天空)**，一束体积光斜射。首发：森林/山路/河桥/村庄/洞穴/神庙(隐藏结局)。规格：分层出图、前景直接 `black silhouette`、统一暖色调别忧郁。

### D · 角色/NPC/怪物（剪影为主）
主角/强盗/怪物**全走黑剪影**（造型清晰无需面部，省海量工时）；NPC 委托头像可稍精细走半身立绘（首发 6–8 个）。主角剪影手持发光武器 = 画面焦点。

### E · UI/图标/特效
6 维数值图标、对话框(淡雅轻量)、分享卡片模板、特效(火星/蒸汽/光束**优先引擎粒子生成少画**)。

## 试做样例（作者用 AI 产图验证，按此顺序）
1. **样例 A（最关键）**：① 一张剑的**基础形制底图** ② 一张**大马士革水纹贴图**（叠加层）③ 把水纹叠到剑底图上看效果 → 验证**"底图 + 工艺贴图叠加"能否做出精致质感、且风格统一**。这是两层模型第二层的核心验证；不成则美术风格或叠加方式要调。
2. **样例 C+D**：森林中景 + 树剪影 + 持剑冒险者剪影(套发光武器) → 看**剪影分层+发光武器焦点**的氛围。
3. **样例 B**：发光熔炉 + 铁砧 + 暗背景 → 看铁匠铺氛围。
> 判断标准不是"单张多精美"，而是**贴图叠到底图上自不自然、多张风格统不统一、发光武器焦点突不突出、暖色氛围对不对**。

- [ ] 造几把极端武器试：全高/偏科(外观10锋利0)/全低
- [ ] 看：不同武器的故事是否真的不一样、且有意思
**算过**：同武器每次故事一致；不同武器故事明显不同；偏科武器翻车出戏剧性。 **存档**：`[S2] 数值驱动故事跑通`
> 🎯 **全项目最关键验证**。故事不够吸引人 = 卖点二有问题，**现在调最省成本**（还没做美术）。过了这步，游戏的"魂"就成立了。

### ✅ S3 · 验证锻造产出不同武器（2–3 天）

- [ ] 能点/拖/滑做基本操作，操作后屏幕实时显示状态变化
- [ ] 一套操作走完能产出带 6 维的武器
**算过**：认真操作 vs 乱敲，产出的武器数值明显不同。 **存档**：`[S3] 锻造产出不同武器`

### ✅ S4 · 把画面串起来（2–3 天）
- [ ] AI 做武器可视化 + 2.5D 场景导航（**先用占位图/色块，别精美美术**）
- [ ] 点工具进对应工序；武器数值变→画面外形跟着变
**算过**：能从场景点进各工序、做完回场景；外形随数据变。 **存档**：`[S4] 画面场景串通`
> 到这步才需要开始认真产美术（第三部分）。之前都用占位图。

### ⭐🔴 S5 · 磨锻打/打磨手感（时间弹性，唯一真难点）
- [ ] 反复试玩，让 AI 调：单次敲击形变量、反馈强度(火星/震动/音效)、打磨手感
- [ ] 每次调完自己玩，凭感觉判断爽不爽
**算过**：你自己玩着觉得"有打铁爽感、想多敲几下"。 **存档**：每次满意就 `[S5] 手感迭代 N`
> 🔴 没有标准答案，靠反复试玩磨。预留充足时间，别指望一次到位。这是最能体现你"手感理解"的地方，值得花时间。

**算过**：走通"做武器→雇NPC看故事"和"做武器→二维码→扫码看故事"。 **存档**：`[S6] 社交闭环打通`
