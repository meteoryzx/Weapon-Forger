# 《打了个铁》协作与 Prompt 工程入门

这份指南给不写代码、但要指挥 Codex 做完整项目的作者使用。目标不是背命令，而是知道每个工具解决什么问题、什么时候需要验收。

## 1. 我们怎么分工

- **作者**：决定体验、取舍和优先级；亲手试玩；用现象描述问题；确认是否通过。
- **Codex**：实现代码与数据；管理测试、Git 和构建；解释风险；提供可点击的验收证据。

作者不需要判断代码写得漂不漂亮，但必须判断玩家能不能理解、操作是否有趣、故事是否真由锻造结果造成。

## 2. 一轮开发只有八步

1. 用 **Issue** 写任务卡：这次要解决什么，什么不做，怎样算完成。
2. 用 **Goal** 绑定当前任务：防止长对话跑偏。
3. 建 **branch**：在不影响正式版本的草稿区工作。
4. 建 **Draft PR**：把半成品和讨论同步到 GitHub。
5. Codex 实现并运行 **CI/测试**。
6. 作者打开 Demo 试玩，按玩家行为验收。
7. Codex review 改动，修复后再请作者确认。
8. 通过后 squash merge 到 `main`，形成一个干净正式存档。

## 3. 常用名词

| 名词 | 大白话 | 本项目里什么时候用 |
|---|---|---|
| `main` | 始终能玩的正式版本 | 只有验收通过的 PR 能进入 |
| Issue | 任务卡/合同 | 每个 2-4 天切片开始前 |
| branch | 独立草稿纸 | 所有修改都在这里发生 |
| commit | 本地存档点 | 每个可描述检查点和离开电脑前 |
| push | 上传 GitHub | 每次 commit 后、换电脑前 |
| Draft PR | 未完成的合并申请 | 第一个检查点上传后立即建立 |
| PR | 正式合并申请 | 测试、review、试玩都通过后 |
| CI | GitHub 自动质检 | 每次 push 自动检查纯逻辑 |
| tag | 不会移动的里程碑标签 | S3a/S3b 等大阶段通过后 |
| Git LFS | 大文件仓库 | PSD、WAV、BLEND 等源资产 |

## 4. 常用安全命令

这些命令通常由 Codex 执行；作者需要理解含义和风险。

```bash
git status
```

查看当前分支和有没有未存档修改。每次开工第一步。

```bash
git switch -c feat/S3a-simulation
```

从当前稳定版本创建一张新草稿纸。

```bash
git add <明确的文件> && git commit -m "[WIP][S3a] 温度规则检查点"
git push
```

把明确的一组修改做成存档并上传。避免不加检查地使用 `git add .`。

```bash
git pull --ff-only
```

只接受不会制造额外合并的远端更新。新电脑接手时使用。

```bash
git revert <提交号>
```

用一个新的反向提交撤销已经进入正式历史的修改，可恢复、可审查。

`git reset --hard` 会直接丢弃本地内容，日常协作禁止使用。必须先有远端副本或 rescue 分支，并得到作者明确批准。

## 5. 一个合格 Prompt 的四部分

```text
Goal：这次要改变的玩家行为。
Context：相关 Issue、文件、现象和参考。
Constraints：模块边界、禁止事项、时间范围。
Done when：自动测试和作者可操作的验收标准。
```

示例：

```text
Goal：实现高温刀坯比低温刀坯更容易发生有效形变。
Context：读取 Issue、AGENTS.md 和锻造研究报告的温度部分。
Constraints：只改纯模拟与测试；禁止 Cocos 依赖；禁止直接增加六维数值。
Done when：相同锤击能量下，高温样本形变量更大；低温样本应力更高；测试可重复。
```

## 6. 换电脑的核心规则

- 两台电脑不能同时开发。
- 离开前关闭 Cocos、运行检查、更新 `CURRENT_TASK.md`、commit、push。
- 新电脑先核对分支和提交，再安装依赖、运行 `doctor`、测试和 H5 构建。
- 聊天记录不是项目真相；GitHub、Issue、PR、`CURRENT_TASK.md` 和决策日志才是。

## 7. Skill、AGENTS 和 Goal 的区别

- `AGENTS.md`：每次都要遵守的项目法律。
- Skill：会重复发生的标准操作流程，例如设备交接或数值审计。
- Goal：当前这一趟工作的目的，完成 PR 后清除。
- Prompt：本次具体要求，不承担长期记忆。
