## 0. 一句话项目定位
玩家扮演铁匠，用**可自由操作的拟真锻造系统**打造武器（卖点一）；每把武器带 **6 维数值**，带着它进入一段**"轻跑团"式冒险故事**（卖点二，选择+武器+世界状态涌现结局）。**性质=作者求职游戏策划岗的作品集**，重设计深度 + 完整可玩核心闭环，不追求商业完整度。

## 2. 与作者协作的铁律（作者的明确要求，必须遵守）
- **作者不写代码**，他是"架构师 + 质检员"。你产出，他靠"行为/效果"验收，不看代码本身。所以**一切要可验证化**：每交付一块，给一个"他能点/能看"的测试（demo/数据打印/录屏），不是"我写完了"。
- **一次只做一个模块/一小步** → 作者验收通过 → git 提交 → 再下一步。**绝不一次做一大坨**。
- **改动前先说计划，等作者点头再动手**。删文件/大改前必须先问。
- **每步都要自检**（见 `AI工作规范.md` §6 自检清单），报错用人话说清"哪里·为什么·打算怎么办"。
- **作者极重视工程规范**（责任划分/低耦合/隔离/规范 git）——这是他作品集的加分项，务必做扎实。

## 3. 绝不能做（作者已论证否决的，别"好心"捡回来）
- ❌ **多 Agent 团队**——现阶段单人单线，不搭。
- ❌ **手搓引擎 / 换 Unity**——用 Cocos Creator（TS）。
- ❌ **真实物理仿真（软体形变）**——锻打/研磨用"分段缩放"伪形变即可。
- ❌ **6 维当"独立能力值"**——它们是**三对物理取舍**（硬↔韧/利↔耐用/轻↔结实），无全满神刀。
- ❌ **武器"预制部件拼装库"**——武器外观是**锻造"打"出来的**（物理外观两层模型），非选零件。
- ❌ **卖点二做成"重跑团"（剧情大分叉）**——只做"轻跑团"（世界状态变量+条件文案），否则内容黑洞、必烂尾。
- ❌ **冒险"单出口"（只分享好友）**——必须双出口（雇NPC自己看 + 分享好友），否则铁匠玩家没即时爽点。
- ❌ **在纸上拍死"具体数值/故事分支"**——那些留到原型里边做边调。

## 4. 核心共识速览（读了能秒懂项目"为什么这么设计"）
- **设计哲学**：受 Noita 影响——"给一套规则系统，让玩家组合出无限可能"。锻造=操作为地基、工艺为配方；故事=6维为接口、内容为配方。
- **锻造系统**：真实工序（选料→加热→锻打→(正火)→淬火→**回火**→研磨→装配）抽象成游戏步骤；**回火是玩家主动定"硬vs韧"取舍的关键**；研磨能改形+定刃角（非只开刃）；折叠/大马士革=花纹/收藏加成+翻车风险，非纯buff。详见全案 §2 + `锻造工艺研究报告.md`。
- **"拟真↔游戏"是可滑动旋钮**，非二选一；平衡点原型里迭代找。**"能离散就不连续"**（旋转固定90°等）。
- **唯一真技术壁垒 = 锻打/研磨实时形变**，用"分段缩放"跨越；其余全是纯逻辑（AI 擅长）。**手感70%靠反馈（火花/音效/数字跳动），非形变精度。**
- **数值三层结构**：物理模拟层(4根基量) → 6维评估/压缩层 → 故事层。6维是"根基量的输出+面向故事的压缩接口"，非最根基。物理模拟层放 Demo 之后（M3 内部升级，接口不变）。
- **平台**：PC 键鼠为主控制方案（Demo，操作最带感）；**核心逻辑与输入层解耦**，保留手机/微信触屏适配可能。Demo 导 H5 验证，不碰微信开发者工具。
